# Обзор проекта

## Назначение

Этот проект служит тестовой площадкой для оценки практических рабочих процессов разработки с помощью AI в Cursor IDE. Цель — понять, как AI-ассистенты могут эффективно сотрудничать с разработчиками в реальном проекте, следуя структурированным правилам и лучшим практикам.

## Рекомендации по использованию AI-ассистента

AI-ассистент в этом проекте должен:

- **Уточнять неясности** — задавать вопросы, когда контекст неясен, вместо предположений
- **Работать итеративно** — вносить небольшие, сфокусированные изменения с объяснениями перед реализацией
- **Уважать существующий код** — никогда не удалять и не перезаписывать без явного подтверждения
- **Оставаться реалистичным** — не изобретать файлы, API или инструменты, которых не существует
- **Общаться ясно** — использовать технический, лаконичный язык без маркетинговых формулировок
- **Поддерживать восстановление** — помогать откатываться к рабочим состояниям при поломках

Рабочий процесс делает акцент на прозрачности, безопасности и итеративном улучшении, а не на быстрых крупных изменениях.

## Текущее состояние

Проект сейчас содержит:
- Простую игру на Python "Угадай число" (`ugadai_chislo.py`)
- Файлы документации с правилами взаимодействия с AI (`AI_RULES.md`)
- Базовое описание проекта (`README.md`)

## Следующие логические шаги

Возможные направления для расширения этого тестового проекта:

1. **Добавить больше примеров кода** — ввести разные типы кода (веб-приложение, API, обработка данных) для тестирования AI-помощи в различных областях
2. **Создать набор тестов** — добавить unit-тесты для проверки изменений кода, созданных AI
3. **Рабочий процесс контроля версий** — протестировать помощь AI при операциях с git и стратегиях ветвления
4. **Упражнения по рефакторингу** — практиковать улучшения и оптимизацию кода с помощью AI
5. **Генерация документации** — протестировать способность AI создавать и поддерживать документацию проекта
6. **Сценарии обработки ошибок** — оценить, как AI помогает отлаживать и исправлять проблемы
7. **Примеры интеграции** — добавить зависимости и внешние библиотеки для проверки понимания AI контекста проекта

Проект может развиваться органически в зависимости от того, какие аспекты разработки с помощью AI требуют наибольшей оценки.
